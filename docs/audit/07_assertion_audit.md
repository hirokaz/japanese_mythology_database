## 6. 監査総括

### 6.1 重要度別の指摘件数(全種別総合)

| 種別 | 高 | 中 | 低 | 計 |
|---|---|---|---|---|
| HYP_ASSERT | 0 | 4 | 7 | 11 |
| 陰謀論寄り | 0 | 0 | 0 | 0 |
| SCH_MIX | 0 | 6 | 2 | 8 |
| 地域伝承断定 | 0 | 1 | 2 | 3 |
| SRC_MISS | 0 | 4 | 1 | 5 |
| **合計** | **0** | **15** | **12** | **27** |

→ **重要度「高」の危険な断定はゼロ**。設計段階から仮説レベル・出典強度を分離するスキーマ設計の効果が現れている。

### 6.2 概観

- **健全な点**:
  - 仮説と事実の分離(hypothesis_layer の運用)
  - 複数説併記の徹底
  - 陰謀論的解釈の不在

- **改善が必要な点**:
  - 神社創建年代の **社伝/学術の二段表記**(SCH_MIX の中)
  - notes 内主張の出典の **括弧書きでの明示**(SRC_MISS の中)
  - 仮説レベルが既に L2-L5 で記録されているものでも、「断定的でない」表現への書き換え推奨

### 6.3 推奨対応

1. **編 TSV 修正**: 神社創建年代の二段表記(主に shrine_master 関連の編行)
2. **マスター修正**: notes 内出典の括弧書き化
3. **hypothesis_master 拡張**: HYP-002, HYP-005 等の source_text_ids 特定

修正は別 issue で実施(監査結果の反映フェーズ)。

### 6.4 監査原則の遵守確認

| 監査原則 | 本監査での適用 |
|---|---|
| 断定しすぎない | ○ 「概ね健全」「多く」等の表現を維持 |
| 神話を単純否定しない | ○ 神話記述自体への批判はせず、伝承と学術の分離を促進 |
| 学術説と社伝を分離 | ○ SCH_MIX で 8 件指摘 |
| 地方伝承を軽視しない | ○ 地方伝承を独立して尊重(監査 #9 で詳述) |
| DB 品質向上を最優先 | ○ 重要度別の優先順位を明示 |

## 7. 全体評価

本DBは設計段階から **仮説と事実の分離が構造化** されており、危険な断定の絶対量は少ない。設計判断(hypothesis_layer の必須化、source_reliability の独立カラム化)が機能している。

中重要度の指摘は主に「表現の二段化(社伝/学術)」と「出典の特定強化」に集中。修正は別 issue でまとまった対応が可能。
