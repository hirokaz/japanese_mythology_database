# 監査フレームワーク

全監査文書(`01_*` 〜 `08_*`)が共通して使用するフレームワーク。

## 1. 問題種別(8 区分)

| 種別 | 略号 | 定義 |
|---|---|---|
| 明確な誤り | `ERR` | 史料・学界の通説と明らかに矛盾する記述 |
| 表記揺れ | `VAR` | 同一エンティティが複数表記で重複登録、または別表記が ID 化されていない |
| 同一神誤統合 | `MERGE` | 別神を同一神として統合、または逆に同一神を別神として分離 |
| 神社関係誤り | `SHR_ERR` | 主祭神・社格・創建伝承の誤り |
| 豪族関係誤り | `CLN_ERR` | 系譜・改姓・関連神社の誤り |
| relation 暴走 | `REL_OVER` | 同体視・習合・archaeologically_linked の過剰適用 |
| 仮説の断定化 | `HYP_ASSERT` | 仮説レベル L1+ の主張が L0(事実)として記述されている |
| 出典不足 | `SRC_MISS` | 出典欄が空、不明確、検証不能 |
| 時代矛盾 | `TIME_ERR` | mythic_time / estimated_historical_time / document_written_time の混入 |
| 地方軽視 | `REGION` | 地方伝承を中央史観に従属させた記述 |
| 学社混同 | `SCH_MIX` | 学術説と社伝・地方伝承の混同 |

## 2. 重要度

| ラベル | 意味 | 対応の緊急度 |
|---|---|---|
| 高 | DB 信頼性に直接影響、誤伝播リスク大 | 即時修正推奨 |
| 中 | 残置すると後続編纂・派生派生物で誤りが拡大 | 次フェーズまでに修正 |
| 低 | 改善余地あるが緊急性なし、将来検討 | 任意 |

## 3. 監査の出力カラム(全文書共通)

```
| 対象 | 問題種別 | 問題内容 | 修正提案 | 根拠 | 推奨出典 | 重要度 |
```

- **対象**: master_id(IZM-XXX, DEI-XXX, SHR-XXX, CLN-XXX, HYP-XXX, RLN-XXXXXX)または対象ファイル名
- **問題種別**: §1 の略号
- **問題内容**: 何が問題か
- **修正提案**: 具体的な修正案(ID付替え、カラム追加、出典追加、複数説併記、仮説レベル変更 等)
- **根拠**: なぜそれが問題か(別の史料、学術通説、考古学的証拠)
- **推奨出典**: 修正時に参照すべき出典(NDL/文化庁/神社本庁/学術論文等)
- **重要度**: 高/中/低

## 4. 監査の動作モード

### 4.1 単独監査(本ディレクトリ)

各監査文書(`01_*` 〜 `08_*`)が **独立に** 既存 DB を検査。指摘のみ。

### 4.2 修正フェーズ(別 issue)

監査結果に基づく修正は **別 issue** を立てて行う。本ディレクトリはあくまで「監査記録」であり、修正は編 TSV / マスター TSV / 関係性 TSV 側で行う。

### 4.3 再監査

修正後、該当監査文書を更新(完了マークまたは「対応済」記入)。

## 5. 監査者の振る舞い(内部規律)

1. **断定しすぎない** — 「明確な誤り」と判断する前に、複数説の可能性を必ず検討
2. **神話を単純否定しない** — 神話を史実と混同しない、ただし神話の文化的価値を否定しない
3. **学術説と社伝を分離** — 学術通説と神社縁起・地方伝承を区別する
4. **地方伝承を軽視しない** — 中央史観の代替として、地方視点を独立に評価
5. **DB 品質向上を最優先** — 編纂者を批判するのではなく、データ構造の改善を優先

## 6. 監査文書のバージョニング

各監査文書の冒頭に:

```
監査日: YYYY-MM-DD
監査対象バージョン: <git commit hash の頭 7 文字>
監査者: Claude(編纂エージェント)
```

を記録(将来運用で標準化)。
