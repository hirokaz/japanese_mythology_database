# 監査・検証ディレクトリ

本ディレクトリは「日本神話・神社・豪族・皇統・祭祀・考古学統合データベース」の **品質を保つための監査文書群** を格納する。

## 監査の役割

このディレクトリの文書は **新規データを生成するものではない**。既存の(または編纂中の)データに対し、

- 明確な誤り
- 表記揺れ・同一神誤統合
- 神社・豪族関係の誤り
- relation の暴走
- 仮説の断定化
- 出典不足・時代矛盾
- 地方伝承の軽視
- 学術説と社伝の混同

を **指摘し、修正提案** することが役割。

## 監査原則

### 1. 監査役の立場

- **新規データを生成しない**(別 issue で実施)
- 指摘 → 修正提案 → 担当(編纂者)が対応 → 再監査
- 複数の解釈が成立する場合、**全て併記** を推奨

### 2. 神話を単純否定しない

- 「史実ではない」と断定するのではなく、「史料種別=神話」「史実性レベル=C」で構造化
- 神話的真実(物語が伝える文化的記憶)は別軸で評価

### 3. 学術説と社伝の分離

- 社伝・神社縁起は `local_tradition` または `official` レベル
- 学術論文は `academic` レベル
- 両者を混在させた断定は監査対象

### 4. 地方伝承の軽視禁止

- 中央史観(古事記・書紀)に偏重しない
- 風土記・郷土史・口碑も独立した史料として扱う
- 「中央視点ではA、地方視点ではB」の併記を推奨

### 5. 仮説の強度を必ず明示

- L0(史料整理) と L4-L5(大胆仮説) を絶対に混同しない
- 仮説関係(syncretized_with、archaeologically_linked)に hypothesis_layer を必須付与

## 監査の出力形式(共通)

各監査は **表形式** が中心。最低以下のカラムを含む:

| 対象 | 問題種別 | 問題内容 | 修正提案 | 根拠 | 推奨出典 | 重要度 |
|---|---|---|---|---|---|---|

ID(IZM-XXX, DEI-XXX 等)で対象を特定。

## 重要度

| ラベル | 意味 |
|---|---|
| 高 | DB の信頼性に直接影響する誤り。即時修正推奨 |
| 中 | 残置すると後続編纂で誤伝播する可能性 |
| 低 | 改善余地あるが緊急性なし |
