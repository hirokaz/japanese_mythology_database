# 仮説レイヤー設計

史料・解釈・学術仮説・地方伝承・民間仮説・大胆仮説を **混同しない** ためのルールを定める。

## 1. レイヤー定義(L0〜L5)

| Level | コード | 種別 | 定義 |
|---|---|---|---|
| L0 | factual | 史料記載の整理 | 仮説ではなく、史料の事実記述 |
| L1 | general_interpretation | 一般的解釈 | 学界・社家で広く認められる読み |
| L2 | academic_hypothesis | 学術仮説 | 複数研究者が言及する推定 |
| L3 | local_tradition_interpretation | 地方伝承解釈 | 地域に伝わる読み・口碑 |
| L4 | folk_theory | 民間仮説 | 在野・愛好家の論 |
| L5 | speculative | 大胆仮説 | 検証困難だが思想的・構造的に興味深い仮説 |

## 2. 各レイヤーの詳細

### L0: factual(史料記載の整理)

- **意味**: 史料に書かれていることを、解釈を加えず整理する。
- **使用ルール**:
  - 古事記・日本書紀に「○○と書かれている」段階の主張のみ。
  - 「Aが書かれているから史実Bだった」と推論したら L1 以上。
- **例**:
  - 「古事記には大国主の異名として八千矛が記される」(L0)
  - 「出雲国造神賀詞は霊亀2年に初奏上された(続日本紀記載)」(L0)

### L1: general_interpretation(一般的解釈)

- **意味**: 学界・社家・主流神道思想で広く認められる解釈。
- **使用ルール**:
  - 一般的な歴史教科書・神社界で標準視される読み。
  - 反論が皆無ではないが、主流である。
- **例**:
  - 「タケミカヅチ=藤原氏祖神」(L1)
  - 「神在祭は出雲大社・佐太神社等で挙行される」(L1)

### L2: academic_hypothesis(学術仮説)

- **意味**: 学術論文で複数研究者が言及する推定。
- **使用ルール**:
  - 査読論文で議論されている。
  - 主流説 vs 少数説 の存在を明示。
- **例**:
  - 「国譲り神話=ヤマト王権による出雲祭祀権収奪の政治史的読解」(L2、津田左右吉以降)
  - 「銅鐸祭祀終焉=国譲り神話への対応」(L2)
  - 「大物主と大国主の同体・別体論」(L2)

### L3: local_tradition_interpretation(地方伝承解釈)

- **意味**: 地域・社家に伝わる読み。学術検証は限定的。
- **使用ルール**:
  - 採録地・採録者を明示。
  - 学術的な検証性とは別に、文化的価値として記録。
- **例**:
  - 「諏訪のミシャグジは縄文以来の信仰」(L3、守矢家伝・民俗学的記述)
  - 「氷川神社の氷川は斐伊川由来」(L3)

### L4: folk_theory(民間仮説)

- **意味**: 在野・愛好家の論考、専門家の批判的検討を経ていない論。
- **使用ルール**:
  - 提唱者と提唱媒体を明示。
  - 学術的根拠の薄さを `notes` に明記。
- **例**:
  - 「ヤマタノオロチ=製鉄渡来民の征服譚」(L4)
  - 「出雲—諏訪—越の翡翠ロード仮説」(L4)

### L5: speculative(大胆仮説)

- **意味**: 検証困難だが思想的・構造的に興味深い仮説。
- **使用ルール**:
  - 検証可能性が著しく低いことを明記。
  - 思想史・神話学的価値として記録。
- **例**:
  - 「縄文海進(海面上昇)記憶仮説」(L5)
  - 「神武東征=弥生人の東進記憶説」(L5)

## 3. 既存「仮説レベル 0-5」との対応

既存の `docs/templates/columns.md` §仮説レベルとの対応:

| 既存仮説レベル | 本書レイヤー |
|---|---|
| 0(仮説ではなく史料記載の整理) | L0(factual) |
| 1(一般的な研究上の解釈) | L1(general_interpretation) |
| 2(複数研究者が言及する推定) | L2(academic_hypothesis) |
| 3(民間伝承・地域伝承としての解釈) | L3(local_tradition_interpretation) |
| 4(大胆な仮説) | L4(folk_theory)or L5(speculative) |
| 5(検証困難だが思想的・構造的に興味深い仮説) | L5(speculative) |

→ 完全互換。本書は名称をコード化し、説明を強化したもの。

## 4. hypothesis_master.tsv の必須カラム

| # | カラム | 必須 | 内容 |
|---|---|---|---|
| 1 | `master_id` | ○ | `HYP-NNN` |
| 2 | `canonical_name` | ○ | 仮説名 |
| 3 | `layer` | ○ | L0〜L5 |
| 4 | `proposer` | △ | 提唱者(複数可、`\|` 区切り) |
| 5 | `source_text_ids` | △ | 初出文献 TXT-ID |
| 6 | `aliases` | △ | 別名・別表現 |
| 7 | `evidence_pro` | △ | 支持証拠の master_id 列(`\|` 区切り) |
| 8 | `evidence_con` | △ | 矛盾する事実の master_id |
| 9 | `status` | △ | 現代の評価(主流/少数派/在野/支持低下 等) |
| 10 | `notes` | △ | 論争点・備考 |

---

## 5. 仮説関係の張り方

### 5.1 supports / contradicts

仮説 → 事実への有向関係。

例:
- `HYP-004(国譲り政治史仮説) → supports → SITE-NN(銅鐸集中埋納)`
- `HYP-002(製鉄民移動仮説) → contradicts → ART-NN(古代製鉄炉不在)`

### 5.2 提唱者

- `HYP-NN → proposed_by → TXT-NN`(該当論文)
- 複数提唱者は複数 relation を立てる
