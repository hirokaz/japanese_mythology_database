# 豪族マスター設計

`docs/master/clan_master.tsv` の構造定義と運用ルール。

## 1. 解決すべき問題

| 問題 | 解決手段 |
|---|---|
| 改姓(中臣→藤原、鴨→賀茂) | `aliases` 列、`renamed_to` 関係 |
| カバネ(連→朝臣)で表記が変わる | `aliases` に併記 |
| 分家・分立(出雲国造→千家・北島) | 別レコード化 + `descended_from` 関係 |
| 同祖関係(土師・菅原・大江=アメノホヒ系) | 共通の `ancestor_deity_id` |
| 中央氏族 vs 地方氏族 | `clan_type` 分類 |

## 2. カラム定義(全 12 列・固定順)

| # | カラム | 必須 | 内容 |
|---|---|---|---|
| 1 | `master_id` | ○ | `CLN-NNN` |
| 2 | `canonical_name` | ○ | 改姓後の最新代表名(藤原氏 等) |
| 3 | `canonical_reading` | ○ | ひらがな |
| 4 | `aliases` | △ | 改姓前・カバネ表記等(`\|` 区切り) |
| 5 | `aliases_reading` | △ | aliases 対応読み |
| 6 | `clan_type` | ○ | 中央豪族/地方豪族/祭祀氏族/技術氏族/渡来氏族 |
| 7 | `ancestor_deity_id` | △ | 祖神の DEI-ID |
| 8 | `related_shrine_ids` | △ | 関連神社 SHR-ID(`\|` 区切り) |
| 9 | `related_region_ids` | △ | 支配/本拠地域 |
| 10 | `related_emperor_ids` | △ | 婚姻・服属関連の皇族 EMP-ID |
| 11 | `peak_period` | △ | 最盛期(時代名) |
| 12 | `notes` | △ | 派生氏族・分立・備考 |

## 3. 運用ルール

### 3.1 改姓と分立の使い分け

| ケース | 扱い |
|---|---|
| 同一氏族が改姓(中臣→藤原) | **同一レコード** + aliases に旧名 + `renamed_to` 関係(時系列) |
| 氏族が分立(出雲国造→千家・北島) | **別レコード** + 親氏族との `descended_from` |
| カバネ変更(連→朝臣) | **同一レコード** + aliases に併記 |
| 同祖の別氏族(土師→菅原・大江) | **別レコード** + 同じ `ancestor_deity_id` |

### 3.2 clan_type 分類

- **中央豪族**: ヤマト王権の中枢氏族(藤原・物部・蘇我・大伴・紀)
- **地方豪族**: 国造系・地方有力氏族(出雲国造・宗像氏・諏訪氏)
- **祭祀氏族**: 祭祀職を世襲(忌部・中臣・卜部・三輪)
- **技術氏族**: 特定技術職(土師・玉作・忌部・服部・秦氏)
- **渡来氏族**: 渡来系(秦・東漢・西文)

複合する場合は `/` で複合表記(例: 「中央豪族/祭祀氏族」)。

### 3.3 祖神(ancestor_deity_id)の扱い

- **記紀・系譜資料に基づく祖神**を記録
- 祖神が複数あれば `|` 区切り
- 祖神が神話的(史実性 C 以下)であることは前提として認識

## 4. サンプル成果物

`docs/master/clan_master.tsv` に **20 氏族** を投入。出雲編で頻出+主要中央豪族を含む。
