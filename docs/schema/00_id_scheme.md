# ID 命名規則

本プロジェクトの全エンティティ・関係性に対する ID 体系を定める。

## 1. 基本形式

```
<PREFIX>-<連番>
```

- PREFIX は **3〜5 文字の英大文字**
- 連番は **3 桁**(マスター)または **6 桁**(関係性)、ゼロパディング
- ハイフン区切り

## 2. プレフィックス一覧

## 3. 連番の運用ルール

### 3.1 採番

- 各 master ファイル内で **連番を厳密に保持**(欠番OK、再利用NG)。
- 新規追加は常に **末尾の最大番号 +1**。
- 関係性は採番管理用 `next_relation_id.txt` 等を併用してもよい(将来検討)。

### 3.2 不可逆性

- 一度割り振った ID は **削除しない**。
- エンティティを統合する場合(同一視が確定した場合)、片方を `merged_into` で参照する形で残す。

例: `DEI-007` 大物主 が `DEI-001` 大国主 と同一視確定した場合、
`DEI-007` は `merged_into=DEI-001` を保持し、レコード自体は残す。

### 3.3 桁あふれ

- 連番が 3 桁(999)を超える場合、**4 桁化**(`DEI-1000`)を許容する。
- 6 桁(`RLN-999999`)を超える場合、**7 桁化**(`RLN-1000000`)を許容。

## 4. Layer A ↔ Layer B の相互参照

### 4.1 既存編 TSV の関連カラムにマスター ID を併記

既存の出雲編 TSV では「関連祭神」「関連神社」などのカラムに **自然言語表記** が入っている。今後は段階的に **マスター ID への参照** へ移行する。

例(現状):
```
関連祭神: 大国主、スサノオ
```

例(移行後):
```
関連祭神: DEI-001|DEI-002
関連祭神_表示名: 大国主|スサノオ
```

### 4.2 移行戦略

- 既存編 TSV を一度に書き換えない(リスク回避)。
- 新規編からマスター ID 参照を採用。
- 既存編は別 issue で段階移行(将来的)。

## 5. 例: 出雲編との対応

| Layer A の関連 | Layer B のマスター ID(予定) |
|---|---|
| 大国主 | `DEI-001` |
| スサノオ | `DEI-002` |
| 出雲大社 | `SHR-001` |
| 出雲国造 | `CLN-001` |
| 国譲り神話 | `MYTH-001` |
| 古事記 | `TXT-001` |
| 荒神谷遺跡 | `SITE-001` |
| 銅鐸祭祀終焉=国譲り対応説 | `HYP-001` |

確定は後続 issue(神名/神社/豪族マスター)で行う。

## 6. ID の命名における禁則

- 半角スペース禁止
- 日本語文字禁止
- 連続ハイフン禁止
- 先頭・末尾ハイフン禁止
