# 神名マスター設計

`docs/master/deity_master.tsv` の構造定義と運用ルール。

## 1. 解決すべき問題

| 問題 | 解決手段 |
|---|---|
| 同一神の別名(大国主=オオナムチ=八千矛=大物主?) | `aliases` 列、`same_as` 関係、`merged_into` |
| 異表記(タケミカヅチ=武甕槌=建御雷=鹿島神) | `aliases` 列で全表記収容 |
| 神仏習合(スサノオ=牛頭天王) | `syncretism` 列 + 別レコード化 + `syncretized_with` 関係 |
| 地方変異(佐太大神=猿田彦?) | `regional_variant` 列 + `regional_variant_of` 関係 |
| 文献ごとの記述差(古事記/書紀/風土記) | `main_text_appearance` 列 + `mentioned_in` 関係 |

## 2. カラム定義(全 16 列・固定順)

| # | カラム | 必須 | 内容 |
|---|---|---|---|
| 1 | `master_id` | ○ | `DEI-NNN`(3桁、不可逆) |
| 2 | `canonical_name` | ○ | 代表表記(古事記表記を基本とする方針) |
| 3 | `canonical_reading` | ○ | ひらがな |
| 4 | `aliases` | △ | 別名・別表記を `\|` 区切り。なければ `-` |
| 5 | `aliases_reading` | △ | aliases に対応する読み(同順) |
| 6 | `category` | ○ | 神格カテゴリ(`/` で複合可)。例: 王権神/国土神 |
| 7 | `gender` | ○ | 男 / 女 / 両性 / 不明 |
| 8 | `main_text_appearance` | ○ | 主要登場文献(`\|` 区切り、TXT-ID 化は将来) |
| 9 | `syncretism` | △ | 神仏習合・同体視。なければ `-` |
| 10 | `regional_variant` | △ | 地方変異名・別系統視点 |
| 11 | `parent_deity_ids` | △ | 親神の DEI-ID(`\|` 区切り) |
| 12 | `consort_deity_ids` | △ | 配偶神(順不同・複数可) |
| 13 | `related_shrine_ids` | △ | 主要関連神社の SHR-ID(後日リンク) |
| 14 | `related_myth_ids` | △ | 主要登場神話の MYTH-ID(後日リンク) |
| 15 | `merged_into` | △ | 統合先 DEI-ID。通常 `-` |
| 16 | `notes` | △ | 論争点・備考 |

## 3. 運用ルール

### 3.1 canonical_name の選定基準

優先順位:
1. **古事記表記**(漢字仮名混じり)を基本
2. 古事記未登場の神は **書紀本文表記**
3. 書紀未登場の神は **出雲国風土記等の風土記表記**
4. 中世神道神(牛頭天王 等)はその初出表記

### 3.2 aliases に含めるもの

- 書紀での別表記(例: 須佐之男命/素戔嗚尊)
- 機能名(例: 大穴牟遅、八千矛)
- 別名(例: 建速須佐之男)
- 後世の別称(例: 諏訪明神)
- 神仏習合での別号(例: 牛頭天王)— ただし重要な習合神は別レコード化推奨

### 3.3 神仏習合の扱い(重要)

**ルール**: 神仏習合での同一視は、原則 **習合先を別レコード**(別 DEI-ID)で立て、`syncretized_with` 関係で結ぶ。

- 例: スサノオ(`DEI-002`)と 牛頭天王(`DEI-030`)は別レコード、`syncretized_with`(L1)
- 理由: 習合期の祭祀・信仰実態が独立して論じられるため

ただし、古事記書紀の異表記レベルの「別名」は aliases にまとめる(例: 須佐之男命/素戔嗚尊は同レコード)。

### 3.4 同体・別体論争の扱い

確定していない同一視(例: 大国主と大物主、佐太大神と猿田彦)は:
- **別レコード** として立てる
- `syncretized_with`(L2 仮説)関係で結ぶ
- `notes` に論争の所在を明記

### 3.5 ID 採番ポリシー

- 神話的・歴史的重要度ではなく、登録順で採番
- ただし、出雲編で頻出する主要神を `DEI-001`〜`DEI-030` に優先確保
- 桁あふれ(>999)は 4 桁化許容

## 4. 既存編 TSV(出雲編)との接続

出雲編の「関連祭神」カラムの自然言語表記を、段階的に DEI-ID に置換する。

例:
```
関連祭神(現状): 大国主、スサノオ
関連祭神_id(将来): DEI-001|DEI-002
```

## 5. サンプル成果物

`docs/master/deity_master.tsv` に 30 件のサンプル(DEI-001〜DEI-030)を投入。
出雲編で頻出する神を中心に、皇統系・宗像系・諏訪系・在来系を含む。
